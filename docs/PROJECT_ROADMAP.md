# Project Roadmap

**Project:** Environmental Operational Business Intelligence Warehouse

**Version:** 1.0

---

# Purpose

This roadmap provides a high-level view of the Environmental Operational Business Intelligence Warehouse project.

It describes the major phases of Version 1.0 from the original business problem through the completed warehouse.

Unlike the Technical Manual, which explains the project, or the Implementation Plan, which details the next development tasks, this roadmap shows the overall direction of the project.

---

# Project Vision

The goal of Version 1.0 is to build and document a complete operational data warehouse based on real business processes.

The finished repository should demonstrate:

- Business understanding
- Relational database design
- SQL development
- ETL processes
- Data validation
- Technical documentation
- Reporting examples

The project emphasizes practical implementation over theoretical design.

# Version 1.0 Definition of Success

Version 1.0 will be considered complete when the repository contains:

- A functioning SQL warehouse.
- Completed staging, dimension, and fact tables.
- Repeatable ETL scripts.
- Validation queries.
- Technical documentation.
- Supporting diagrams.
- Example reporting.
- Portfolio-ready presentation of the project.

The objective is not to build every possible feature.

The objective is to build a complete, well-documented operational data warehouse that accurately represents the current implementation of the project.
